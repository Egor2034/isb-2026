####  Вариант 1  
<details> <summary>Проверка целостности файлов с использованием хеш-функций</summary>

### Цель работы

Изучить принципы работы криптографических [хеш-функций](https://habr.com/ru/post/534596/) и реализовать механизм проверки целостности файлов.

---

### Краткие теоретические сведения

Хеш-функция — это алгоритм преобразования данных произвольной длины в строку фиксированного размера. Криптографические хеш-функции используются для:
- контроля целостности данных;
- хранения паролей;
- цифровых подписей;
- аутентификации сообщений.

Для проверки целостности файла вычисляется его хеш-сумма. При изменении содержимого файла изменяется и его хеш.

---

### Задание

Разработать программу, выполняющую:
1. Вычисление хеша файла.
2. Сохранение контрольной суммы.
3. Проверку целостности файла.
4. Сравнение текущего и сохранённого хеша.

---

### Требования к программе

Программа должна:
- поддерживать выбор файла;
- использовать алгоритм SHA-256;
- сохранять хеш в отдельный файл;
- выполнять повторную проверку файла;
- выводить результат проверки.

---

### Порядок выполнения работы

1. Изучить принцип работы SHA-256.
2. Реализовать вычисление хеша файла.
3. Создать тестовый файл.
4. Вычислить и сохранить хеш.
5. Изменить содержимое файла.
6. Выполнить повторную проверку.
7. Проанализировать результаты. 

---

### Контрольные вопросы

1. Что такое хеш-функция?
2. Для чего применяется контроль целостности?
3. Что такое коллизия?
4. Почему изменение одного символа изменяет весь хеш?
5. Чем SHA-256 отличается от MD5?

#### Вариант 2  
<details> <summary>Исследование лавинного эффекта хеш-функций</summary>

### Цель работы

Исследовать лавинный эффект криптографических хеш-функций.

---

### Краткие теоретические сведения

Лавинный эффект — свойство криптографического алгоритма, при котором минимальное изменение входных данных вызывает значительное изменение результата хеширования.

Для качественной [хеш-функции](https://habr.com/ru/post/534596/) изменение одного бита входных данных должно изменять примерно половину бит выходного значения.

---

### Задание

Необходимо:
1. Вычислить хеш исходной строки.
2. Изменить:
   - один символ;
   - один бит;
   - регистр буквы.
3. Выполнить повторное хеширование.
4. Сравнить результаты.

---

### Требования к программе

Программа должна:
- отображать исходный и изменённый хеш;
- подсчитывать количество изменённых бит;
- вычислять процент различий;
- формировать таблицу результатов.

---

### Порядок выполнения работы

1. Выбрать исходную строку.
2. Вычислить её хеш.
3. Последовательно изменять данные.
4. Сравнить результаты хеширования.
5. Провести не менее 10 экспериментов.
6. Сделать выводы о качестве алгоритма.

---

### Контрольные вопросы

1. Что такое лавинный эффект?
2. Почему лавинный эффект важен?
3. Какие хеш-функции обладают хорошим лавинным эффектом?
4. Как оценить качество хеш-функции?
5. Что происходит при изменении одного бита входных данных?

### Краткие теоретические сведения

HMAC обеспечивает:
- контроль целостности;
- подтверждение подлинности сообщения.

---

### Задание

4. Обнаружение изменения данных.

---

### Требования к программе

Программа должна:
- использовать HMAC-SHA256;
- поддерживать ввод секретного ключа;
- проверять подлинность сообщения;
- обнаруживать изменение данных.

---

### Порядок выполнения работы

2. Реализовать вычисление HMAC.
3. Передать сообщение и код аутентификации.

### Цель работы

Исследовать вероятность возникновения коллизий и изучить атаку «парадокс дней рождения».

---

### Краткие теоретические сведения

Коллизия — ситуация, при которой разные входные данные дают одинаковый результат хеширования.

Вероятность коллизий возрастает при уменьшении длины хеша.

---

### Задание

Разработать программу, выполняющую:
1. Генерацию случайных строк.
2. Вычисление укороченного хеша.

4. Сбор статистики по количеству попыток.

---

### Требования к программе

Программа должна:
- автоматически генерировать данные;
- выполнять поиск коллизий;
- поддерживать длину хеша:
  - 8 бит;
  - 12 бит;
  - 16 бит;
- выводить найденные коллизии.

---

### Порядок выполнения работы

1. Реализовать генератор случайных строк.
2. Выполнить вычисление хеша.
3. Укоротить хеш.
